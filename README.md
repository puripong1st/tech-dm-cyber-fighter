# Cyber Fighter: ศึกประลองวิชาดิจิทัล

เว็บสื่อการเรียนรู้สำหรับนักเรียนชั้นมัธยมศึกษาปีที่ 3 ในหัวข้อการใช้เทคโนโลยีสารสนเทศอย่างปลอดภัย ผู้เรียนต่อสู้กับบอสด้วยการตอบคำถาม และครูติดตามผลผ่านแดชบอร์ดได้

## จุดเด่น

- เกมตอบคำถามแบบ RPG แบ่งเป็นด่านและบอส
- คลังคำถาม 75 ข้อ แต่สุ่มเล่นเพียง 25 ข้อต่อรอบ โดยคงสัดส่วน 3 / 4 / 4 / 4 / 10 ข้อ
- ทุกการสุ่มมีโจทย์ระดับการจำ การเข้าใจ และการประยุกต์ใช้ตาม Bloom's Taxonomy
- หน้าจอภาษาไทยที่เน้นการอ่านง่าย ทั้งคำถาม ตัวเลือก และ HUD
- แสดงด่าน/ข้อปัจจุบัน และเลือกคำตอบได้ทั้งคลิกหรือกด `A`–`D`
- เฉลยคำตอบพร้อมเหตุผลหลังตอบผิด เพื่อให้ผู้เรียนทบทวนก่อนลองตอบใหม่
- สรุปหัวข้อที่ตอบพลาด และ Badge ความสำเร็จเมื่อจบภารกิจ
- ระบบ HP, AP, คอมโบ และทักษะพิเศษ
- บันทึกเซสชันที่กำลังเล่นไว้ในเบราว์เซอร์ และบันทึกคะแนนลง Supabase ได้
- แดชบอร์ดครูสำหรับค้นหา กรอง และสรุปผลคะแนน
- รองรับหน้าจอมือถือและการลดการเคลื่อนไหวตาม `prefers-reduced-motion`

## วิธีเล่นสำหรับนักเรียน

1. กรอกชื่อ ห้อง และเลขที่ จากนั้นเลือกตัวละคร
2. อ่านการ์ด “พร้อมเริ่มภารกิจไหม?” เพื่อเข้าใจ HP, AP และการเรียนรู้จากเฉลย
3. ตอบคำถามด้วยการคลิกตัวเลือก หรือกด `A`, `B`, `C`, `D`
4. ตอบถูกเพื่อโจมตีบอส สะสม AP และใช้ทักษะเมื่อจำเป็น
5. หากตอบผิด เกมจะแสดงคำตอบที่ถูกและคำอธิบาย ก่อนให้ลองตอบข้อนั้นอีกครั้ง

## ปรับคลังคำถาม

แก้ไข `game_questions.js` โดยแต่ละข้อมีรูปแบบดังนี้:

```js
{
  header: "STAGE 1 | การเข้าใจ",
  question: "คำถามของคุณ",
  choices: ["ตัวเลือก ก", "ตัวเลือก ข", "ตัวเลือก ค", "ตัวเลือก ง"],
  answer: 1,
  explanation: "เหตุผลสั้น ๆ ที่อธิบายว่าทำไมตัวเลือก ข จึงถูกต้อง"
}
```

`answer` เริ่มนับจาก `0` (ตัวเลือกแรก) ถึง `3` (ตัวเลือกสุดท้าย) ส่วน `explanation` ใช้แสดงเหตุผลหลังตอบผิด ควรใส่ทุกข้อเพื่อให้เฉลยมีคุณค่าต่อการเรียนรู้มากที่สุด

## เริ่มใช้งานในเครื่อง

ต้องมี Node.js จากนั้นสร้างไฟล์ `.env` โดยคัดลอกจาก `.env.example` และกำหนดค่า:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
TEACHER_PASSCODE=choose-a-strong-passcode
```

สั่งสร้างไฟล์สำหรับใช้งาน:

```bash
npm run build
```

ไฟล์ที่พร้อมเผยแพร่อยู่ในโฟลเดอร์ `dist/` หากยังไม่ตั้งค่า Supabase เกมยังเล่นในเบราว์เซอร์ได้ แต่จะไม่บันทึกคะแนนขึ้นคลาวด์

## เผยแพร่บน Vercel

ตั้งค่า Build Command เป็น `npm run build` และ Output Directory เป็น `dist` จากนั้นเพิ่ม `SUPABASE_URL`, `SUPABASE_ANON_KEY` และ `TEACHER_PASSCODE` ใน Environment Variables ของโครงการก่อน Deploy

> ควรใช้ Supabase Anon Key ร่วมกับ Row Level Security และนโยบายที่จำกัดสิทธิ์ให้เหมาะสม ไม่ควรใส่ Service Role Key ในเว็บฝั่งผู้ใช้

## ตรวจสอบก่อนเผยแพร่

```bash
npm run build
```

ทดสอบการลงทะเบียน การตอบถูก/ผิด การกด `A`–`D` การเปิดเฉลย การกลับมาเล่นต่อ และการบันทึกคะแนนบนอุปกรณ์ที่ใช้งานจริง
