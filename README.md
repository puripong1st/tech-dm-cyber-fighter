## 🛠️ รายละเอียดกลไกการทำงานภายใน (Internal Mechanics)

### 1. ระบบจัดการสไลด์เชิงโต้ตอบ (`presentation.html`)
- **การเรนเดอร์**: ใช้ CSS Flexbox แบบ Responsive ครอบทับแต่ละหน้าสไลด์ (`.slide`) โดยจะถูกซ่อนการแสดงผลไว้ผ่านคลาส `.active` ซึ่งควบคุมจาก JavaScript
- **ระบบกล่องสมบัติความรู้ (Interactive Elements)**: มีองค์ประกอบเชิงโต้ตอบสไตล์ Roblox ที่ผู้ใช้สามารถคลิกเพื่อกระตุ้นแอนิเมชันคลี่คลายเนื้อหา/เปิดเผยคำถามลับ (Click-to-reveal) เพื่อให้นักเรียนมีส่วนร่วมตอบคำถามก่อนคุณครูเฉลย

### 2. กลไกในเกมตอบคำถามต่อสู้กับบอส (`cyber_fighter.html`)
- **ระบบสลับตัวเลือกและคลังข้อสอบ (Anti-Cheat Exam Shuffling)**:
  - เมื่อเริ่มต้นรอบเล่นหรือด่านใหม่ อาร์เรย์ของคำถามจะผ่านขั้นตอนการสลับค่าแบบสุ่มด้วยอัลกอริทึม **Fisher-Yates Shuffle** ใน JavaScript
  - ตัวเลือกคำตอบ A, B, C, D ของแต่ละข้อจะถูกจัดสลับตำแหน่งปุ่มตัวเลือกใหม่ทุกครั้ง เพื่อป้องกันการลอกคำตอบของนักเรียนที่นั่งข้างกันอย่างสมบูรณ์แบบ
- **ระบบจดจำสถานะป้องกันการรีเฟรชหน้าจอ (Session Recovery Engine)**:
  - ทันทีที่การประลองเริ่มต้นขึ้น ข้อมูลของผู้เล่น (ชื่อ, นามสกุล, เลขที่, ห้อง, เลือดของบอส/ผู้เล่น, ข้อที่ทำถึงล่าสุด) จะถูกเขียนลงหน่วยความจำของบราวเซอร์ (`localStorage`) ทุกครั้งที่มีการเลือกตอบข้อใดข้อหนึ่ง
  - หากนักเรียนมือเผลอไปกดปุ่ม Refresh (F5) หรือเกิดสัญญาณเน็ตดับกลางคัน หน้าจอจะแสดงปุ่มเปิดรับสถานะเดิมกลับคืนมา **"เล่นต่อจากเดิม ⚔️"** เมื่อคลิกจะไปดึงสถานะจาก `localStorage` มาประกอบข้อมูลหน้าสเตจเกมใหม่ให้สามารถประลองต่อในข้อที่ค้างไว้ได้ทันทีโดยไม่ต้องเริ่มต้นนับหนึ่งใหม่
- **ระบบอัปเดตคะแนนสะสมอัจฉริยะ (Upsert Record Logic)**:
  - ในกรณีที่นักเรียนประสงค์จะแก้ตัวเพื่อทำคะแนนใหม่ ระบบจะเช็กความเข้ากันได้ผ่านชื่อ นามสกุล เลขที่ และห้องเรียน หากพบข้อมูลเดิมอยู่ในระบบ Supabase คลาวด์เรียบร้อยแล้ว ตัวแอปพลิเคชันจะตัดสินใจเรียกคำสั่งอัปเดตแบบ **Upsert (Update-or-Insert)** เขียนบันทึกคะแนนใหม่ทับคะแนนเดิม *เฉพาะกรณีที่คะแนนรอบล่าสุดสูงกว่ารอบที่บันทึกไว้ในคลาวด์เท่านั้น* พร้อมกับทำการบวกจำนวนครั้งที่พยายามเล่นเพิ่มขึ้น 1 ครั้ง (`attempts + 1`) เพื่อไม่ให้ข้อมูลนักเรียนเกิดการซ้ำซ้อนในแดชบอร์ด

### 3. ระบบจัดการคะแนนและออกรายงานของคุณครู (`teacher_dashboard.html`)
- **รหัสผ่านควบคุมความปลอดภัย (Passcode Gating)**:
  - ปิดกั้นหน้าการเข้าถึงแดชบอร์ดด้วยหน้าต่างตรวจรหัสผ่าน (Passcode Gate) ในระดับ Client-side ซึ่งจะนำค่ารหัสผ่านจริงมาใช้งานจากตัวแปรสภาพแวดล้อม `TEACHER_PASSCODE` เพื่อไม่ให้มีรหัสผ่านสำคัญรั่วไหลลงไปในซอร์สโค้ดของระบบ
- **การซิงโครไนซ์ข้อมูล Real-time Database**:
  - เมื่อระบบตรวจสอบว่าตัวแปรสภาพแวดล้อม Supabase ถูกต้อง ระบบจะทำการ Subscribed เข้าสู่ระบบ Real-time (ช่องสัญญาณ `postgres_changes` บนคลาวด์) ทำให้ทุกครั้งที่มีเด็กกดส่งคะแนนสอบจากหน้าจอเกม ข้อมูลในตารางหลังบ้านของคุณครูจะแสดงผลและคำนวณคะแนนใหม่โดยไม่ต้องกดโหลดหน้าเว็บใหม่
- **การประมวลผลคะแนนและการกรอง**:
  - แดชบอร์ดจะรันการกรองคัดลอกข้อมูลเฉพาะตามระดับห้องเรียน (ม.3/1 ถึง ม.3/15) ค้นหารายชื่อนักเรียนด้วยแถบค้นหา และสามารถสลับการเรียงลำดับคะแนน (จากมากไปน้อย หรือ น้อยไปมาก) รวมถึงเวลาในการทำข้อสอบและคะแนนเก็บ (สัดส่วนคะแนนที่นำไปแปลงเปรียบเทียบเต็ม 10 คะแนน)
- **การพิมพ์รายงาน PDF**:
  - นำเข้าไลบรารี `html2pdf.js` มาช่วยดึงเนื้อหาจากแท็ก HTML Table ฝั่งแดชบอร์ดมาแปลงเป็นเวกเตอร์ PDF โดยรองรับการดาวน์โหลดไฟล์ทันทีโดยจัดระเบียบฟอนต์ Chakra Petch บนหน้าเอกสารให้เรียบร้อยสวยงาม

### 4. ระบบจัดการตัวแปรสภาพแวดล้อมและขั้นตอนการ Build
- **เหตุใดจึงใช้ระบบ Build-time Injection?**:
  - เนื่องจากโปรเจกต์นี้ทำงานฝั่งไคลเอนต์ (Static Frontend HTML/JS) หากเราทำการฮาร์ดโค้ด `SUPABASE_URL` และ `SUPABASE_ANON_KEY` ทิ้งไว้ในไฟล์ ทุกคนที่เข้าหน้าเว็บจะสามารถเข้าถึงคีย์เหล่านี้ได้โดยตรง และเสี่ยงต่อการนำไปเขียนโปรแกรมยิงทำลายตารางข้อมูล
  - เพื่อแก้ปัญหานี้ ซอร์สโค้ดใน Git จะถูกลบค่าคีย์เชื่อมต่อจริงออกทั้งหมด และแทนที่ด้วยคำจำลอง (`SUPABASE_URL_PLACEHOLDER` / `SUPABASE_ANON_KEY_PLACEHOLDER`)
  - ไฟล์ [`build.js`](file:///c:/Users/aunkh/OneDrive/Desktop/%E0%B8%81%E0%B8%B2%E0%B8%A3%E0%B9%83%E0%B8%8A%E0%B9%89%E0%B9%80%E0%B8%97%E0%B8%84%E0%B9%82%E0%B8%99%E0%B9%82%E0%B8%A5%E0%B8%A2%E0%B8%B5%20%E0%B8%AA%E0%B8%B2%E0%B8%A3%E0%B8%AA%E0%B8%99%E0%B9%80%E0%B8%97%E0%B8%A8/build.js) จะคอยอ่านตัวแปรสภาพแวดล้อมจากระบบของ Vercel (หรือไฟล์ `.env` เมื่อรันบนเครื่อง) มาสวมสวมแทนที่คำจำลองดังกล่าว จากนั้นเขียนไฟล์ HTML ผลลัพธ์ที่พร้อมใช้งานจริงลงไปที่โฟลเดอร์ผลการสร้าง (`dist/`) ซึ่งจะถูกนำไปเสิร์ฟออนไลน์ ช่วยป้องกันการนำคีย์ไปเก็บไว้ในประวัติ Git ของคุณครู

ใช้โครงสร้างคำสั่ง SQL เพื่อสร้างตารางบันทึกสถิตินักเรียนบนระบบ Cloud:

```sql
-- สร้างตารางสำหรับจัดเก็บคะแนนประลอง
create table cyber_fighter_scores (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  surname text not null,
  student_number integer not null,
  room text not null,
  score integer not null,
  attempts integer not null,
  play_time integer default 0, -- เวลาทำข้อสอบจริง (หน่วยเป็นวินาที)
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- เปิดใช้ช่องทางดึงข้อมูลแบบอัปเดตเรียลไทม์ (Real-time DB Channel)
alter publication supabase_realtime add table cyber_fighter_scores;

-- ปิดการใช้งาน RLS ชั่วคราวสำหรับการยิงส่งคะแนนจากไคลเอนต์โดยตรง (กรณีไม่ได้ทำ Auth)
alter table cyber_fighter_scores disable row level security;
```

## 🚀 คู่มือการตั้งค่าเพื่อนำไปใช้งานจริง (Setup & Deployment)

### 1. วิธีการรันและปรับแต่งบนเครื่องส่วนตัว (Local Development)
1. ติดตั้ง Node.js ลงในคอมพิวเตอร์ของคุณ
2. คัดลอกโปรเจกต์นี้มาไว้ในเครื่อง
3. สร้างไฟล์ชื่อ `.env` ไว้ที่โฟลเดอร์หลักสูงสุดของโปรเจกต์ และใส่ข้อมูลคีย์จำลองของ Supabase และรหัสผ่านแดชบอร์ดดังนี้:
   ```env
   SUPABASE_URL=https://your-project-id.supabase.co
   SUPABASE_ANON_KEY=your-actual-anon-key-here
   TEACHER_PASSCODE=teacher123
   ```
4. ติดตั้งทรัพยากรและสั่งรันคำสั่งบิวด์ผลลัพธ์ผ่าน Terminal:
   ```bash
   npm run build
   ```
5. เปิดไฟล์ผลผลิตหลักจากโฟลเดอร์ `dist/index.html` บนบราวเซอร์เพื่อเริ่มรันงานแบบออฟไลน์

### 2. วิธีการดีพลอยเว็บไซต์ขึ้นระบบ Vercel (Deployment)
เมื่อทำการนำซอร์สโค้ดของโปรเจกต์นี้ขึ้นไปเก็บใน GitHub ของคุณเรียบร้อยแล้ว ให้ดำเนินขั้นตอนดังต่อไปนี้:
1. เข้าสู่ระบบ Vercel และกดปุ่ม **"Add New"** > **"Project"**
2. เลือกเชื่อมเข้ากับคลังข้อมูล GitHub ที่เก็บโปรเจกต์นี้
3. หน้าจอคอนฟิกการนำเข้าโปรเจกต์ (Project Configuration) ให้ระบุข้อมูลระบบดังนี้ (Vercel จะดึงข้อมูลจากไฟล์ `vercel.json` มาใช้แบบออโต้ แต่อาจตรวจทานอีกครั้งได้):
   - **Framework Preset**: Other
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. คลิกส่วนขยาย **"Environment Variables"** และตั้งค่าตัวแปรดังนี้:
   - ชื่อตัวแปร: `SUPABASE_URL` / ค่า: ใส่ URL โครงการ Supabase ของคุณ
   - ชื่อตัวแปร: `SUPABASE_ANON_KEY` / ค่า: ใส่ Anon API Key โครงการ Supabase ของคุณ
5. กดปุ่ม **"Deploy"** รอขั้นตอนบิวด์ของระบบคอมไพล์งานประมาณ 1 นาที จะได้ลิงก์หน้าเว็บหลักสำหรับใช้งานในห้องเรียนจริงทันที
